# テスト規約（共通）

この文書は、`python/tests/` 配下のテストを対象にします。

## 1. 対象と目的
この文書は、`python/tests/` 配下のテストを対象にします。

## 2. 配置と分類
- テストは `python/tests/` に集約します。
- 分類は次の通りです。
	- `python/tests/base/`: 単体テスト（高速）。
	- `python/tests/Algorithms/`: 数値アルゴリズムの検証（中〜高コスト）。
	- `python/tests/neuralnetwork/`: NN 系の検証（小〜中コスト）。
- 大規模ケースは `case` や `*_large` の名称で明示します。

## 3. 実行方法
- **pytest を基本**にし、一括実行を標準とします。
- テストは `python/tests/` を対象に収集します。
- `if __name__ == "__main__":` による単体実行は**補助的に許可**します。

### 3.1 標準出力を表示する
pytest は既定で標準出力を抑制するため、計算結果のログを必ず見たい場合は `-s` を使います。

- ファイル単位: `pytest -q -s python/tests/Algorithms/test_slq.py`
- 全体: `pytest -q -s`

### 3.2 `python file.py` での補助実行
`python path/to/test_xxx.py` は pytest を起動しないため、`test_` 関数は自動では実行されません。

- どうしても単体で回したい場合は、`if __name__ == "__main__":` から補助関数を呼び出して良いものとします。
- この補助実行でも、**計算結果は標準出力へ 1 行 JSON**で出力します。

## 4. 想定解と標準出力ログ
- **想定解のあるテストは、必ず想定解を用意**します。
- その想定解を **標準出力に出力**し、結果と比較できるようにします。
- 1 行 1 レコードの辞書出力を基本にします。

### 4.1 計算結果の標準出力
数値アルゴリズムのテストは、失敗時の原因切り分けに「具体的な数値」が重要です。

- **原則**: テストは計算結果を標準出力へ出します（pytest の表示に乗るようにします）。
- **推奨形式**: 1 行 1 JSON（JSONL と同じ単位）で出力します。
- **推奨キー**:
	- `case`: テスト名（必須）
	- `expected`: 想定値（想定解がある場合）
	- `actual`: 実測値（想定解がある場合）
	- `rel_err` / `abs_err`: 誤差
	- `seed`: 乱数 seed（乱数を使う場合）
	- `dim` / `m` / `s` / `g` など: 条件（チューニング/再現に必要なもの）
- **注意**:
	- 標準出力は可読性を優先し、巨大な配列や長文は出しません（必要なら stats のみ）。
	- 基本的に `print(json.dumps({...}, ensure_ascii=False))` を用います。
- ログ保存は **実行ごとのディレクトリ**にまとめます。
- `pytest.jsonl` には **JSON オブジェクトのみ**を書き込みます。
- 推奨キー例:
	- `case`: テスト名
	- `expected`: 想定解
	- `expected_norm`: 想定解のノルム
	- `actual`: 実測値
	- `rel_err`: 相対誤差
	- `res_norm`: 残差ノルム

## 5. 乱数・大規模テスト
- 乱数を使う場合は **固定 seed** を使用します。
- 悪条件・大規模ケースは **明示的に区別**し、ログに条件を出力します。

## 6. 禁止事項
- 本体モジュール内の `if __name__ == "__main__":` にテストを書きません。
- 例外のみを根拠にせず、**既知解・残差・反復回数**で検証します。
