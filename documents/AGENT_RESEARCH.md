# コーディング Agent に関する調査メモ

このドキュメントは本プロジェクト向けに「コーディング Agent（自己主導でコードや文書の修正・生成を行うエージェント）」の研究動向と、実運用で注意すべき設計上の要点をまとめたメモです。

要点:

- 役割分担: 研究文献では「Planner（タスク分解）」「Executor（実コード生成・変更）」「Verifier（テスト・静的解析）」という三層構成が多く見られます。これを分離すると安全性と検証性が向上します。
- 安全性: 自動コミットや自動修正は必ず検証済み artefact のみ許可する。チェンジは PR ベースで人工レビューを必須にするオプションが推奨されます。
- トレーサビリティ: どのエージェントがいつ何をしたか（プロンプト、決定根拠、テスト結果）を必ずログ化します。
- 人間との協調: エージェントは提案→要約→差分→手動承認 の流れを提供し、自律的に main を直接変更しない設計が安全です。
- モジュール化: エージェント機能は小さな再利用可能モジュール（lint-fix, doc-fix, design-merge, worktree-manager 等）に分けると運用が容易です。

参考（ローカル参照・実務的知見）:

- 本リポジトリ内の `documents/` と `scripts/tools/` にある自動化スクリプト群は、現時点でのベースライン（静的解析集計、類似検出、リンク監査、ワークツリー管理）を満たしています。
- 学術的には、大規模言語モデルを利用したコード生成研究は、「ヒューリスティックなプランニング」と「検証ループ（テスト・静的チェック）」の反復が重要と報告されています。

結論:

まずは現行のスクリプト群をモジュール化し、Agent の各役割（Plan/Exec/Verify）を明確にするディレクトリ構成を採用するのが実用的です。本ドキュメントとともに、下記のディレクトリ構成案を提案します。
