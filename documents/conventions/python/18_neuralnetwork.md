# ニューラルネットワーク規約

## 要約
- `python/jax_util/neuralnetwork/` を対象にします。
- まずは **プロトコルとクラス**を基準に設計を整理します。
- 役割を分離し、拡張の入口を明確にします。

## 規約
### 1. 目的とスコープ
- この規約は、`python/jax_util/neuralnetwork/` 配下の実装を対象にします。
- 現状の目的は **前向き計算（forward）** の設計整理です。
- 学習は `train.py` に集約し、損失はプロトコルとして扱います。

### 2. 基本プロトコル
- `protocols.py` に **最小のプロトコル**を置きます。
- `Ctx` は **固定状態**、`Carry` は **更新状態**です。
- `NeuralNetworkLayer` は `carry` と `ctx` を受け取り、`carry` を返します。

### 3. 役割分離
- `protocols.py` は **契約（Protocol）** だけを定義します。
- `layer_utils.py` は **具体的な層・初期化関数**を置きます。
- `neuralnetwork.py` は **ネットワーク全体の構成と forward** を持ちます。
- `__init__.py` は **公開 API** のみをエクスポートします。

### 4. 型と形状
- `Vector` / `Matrix` の型エイリアスを使います。
- バッチ軸は **プロジェクト共通規約**に従います（`documents/coding-conventions-project.md` の 5.2）。
- 層の入出力は `Matrix` を基本にします。

### 5. コンテキストの扱い
- `Ctx` は **層の共有情報**を保持します。
- `Carry` は **層ごとに更新される状態**を保持します。
- 役割が不明確な値は `Ctx` と `Carry` を分けてください。

### 6. 初期化・ファクトリ
- 初期化は `layer_utils.py` に置きます。
- ネットワークの生成は `neuralnetwork.py` のファクトリ関数にまとめます。
- 乱数キーは **受け渡しで管理**し、副作用を避けます。

### 7. ICNN の扱い
- ICNN 用の重みは **非負性**を保つ設計を採用します。
- 非負制約の処理は **層の内部**に閉じ込めます。
- 解析・拡張を見据え、ICNN と標準 NN の分岐は最小限にします。

### 8. 例外・エラーハンドリング
- 不正な `network_type` は `ValueError` で明示的に拒否します。
- 例外メッセージは **簡潔で丁寧**に書いてください。

### 9. 拡張の方針
- まずは「層・文脈・キャリー」の三点を崩さずに拡張します。
- 追加のネットワーク型は **新しい Ctx / Carry** を用意して対応します。
- 既存の forward 仕様は維持します。

### 10. forward とデータフロー
- `NeuralNetwork.__call__` は **入力 `Matrix` を受け取り `Matrix` を返す**ことを統一します。
- forward 内では **`Ctx` を先に構築し、`Carry` を段ごとに更新**します。
- `carry` の更新以外の副作用は持ちません。

### 11. 初期状態の生成
- 初期状態は `state_initializer`（もしくは同名の関数）に集約します。
- 初期状態は **`Ctx` と `Carry` の両方**を返す設計にします。
- 初期状態の作成に必要な情報は **引数で明示**し、グローバル参照を避けます。

### 12. 層の設計ルール
- 層は `NeuralNetworkLayer` を満たし、`carry` と `ctx` を受け取ります。
- 入力・出力の形状は **`Matrix` のバッチ軸**で統一します。
- `Carry` には **次段に必要な最小情報**のみを持たせます。
- 活性化関数は **`layer_utils.py` 内で受け取る**設計にします。
- forward 内では **同じ活性化関数を繰り返し生成しない**ようにします。
- 未対応の活性化は **`ValueError`** で拒否します。
- ICNN の非負制約は **層の内部**で確実に維持します。
- それ以外の制約は **入力前処理**ではなく **層内で処理**します。
- 変換（例: `abs` / `softplus`）は **1 箇所に集約**します。

### 15. 設計の検証ポイント
- forward は **純粋関数的**に振る舞うことを前提とします。
- `Ctx` / `Carry` の責務が曖昧な場合は **分離を優先**します。
- 新しいネットワーク型追加時は **プロトコルと役割分離が崩れていないか**を確認します。

### 16. NN を実際に動かすまでのワークフロー
1. **入力の形状を決める**
	- 入力は `Matrix` を前提にします。
	- 例: `x.shape == (in_dim, batch)` を前提にします。

2. **ネットワーク型を決める**
	- `standard` か `icnn` を明示します。
	- 分岐は `neuralnetwork.py` のファクトリに集約します。

3. **活性化関数を選ぶ**
	- 文字列で指定し、`layer_utils.py` 側で解釈します。
	- 未対応の活性化は `ValueError` で拒否します。

4. **乱数キーを用意する**
	- 乱数キーは **外部から渡す**設計です。
	- ファクトリはキーを分割し、内部で使います。

5. **ネットワークを構築する**
	- `build_neuralnetwork` のようなファクトリで作成します。
	- 構築時に `layer_sizes` と `activation` を渡します。

6. **初期状態を作成する**
	- `state_initializer` で `Ctx` / `Carry` の初期状態を得ます。
	- 初期化に必要な情報は引数で明示します。

7. **forward を実行する**
	- `NeuralNetwork.__call__` に `Matrix` を渡します。
	- forward 中は `Ctx` を固定し、`Carry` を更新します。

8. **出力の検証を行う**
	- 形状と dtype が想定どおりか確認します。
	- 必要なら最小のテストを追加します。

### 17. 学習の最小設計
- 学習ループは `train.py` に集約します。
- 損失は `protocols.py` の **プロトコル**として定義します。
- 損失の実体は **テスト側で生成**します。
- 最適化器は `train.py` で受け取り、外部から注入します。
- `LossFn` は **`Params` と `Matrix` を受ける**形に統一します。
- `LossFn` の **静的な情報はクロージャー**で閉じ込めます。
- 学習ループは **`jax.lax.scan`** を基本とします。
- `train_loop` は **`batch: Matrix`** を固定し、`num_steps` で反復します。
