# コーディング規約（共通）

この文書は、`python/` と `jupyter/` を含むリポジトリ全体の共通規約をまとめます。

## 1. 基本方針
- 読みやすさと保守性を最優先にします。
- 複雑な分岐は避け、シンプルな実装を心がけます。
- 依存関係は必要最小限に保ち、目的が明確な追加のみ許容します。

## 2. 命名
- 役割が直感的に伝わる名前にします。
- 省略は最小限にし、意味が曖昧になる略称は避けます。

## 3. コメント
- コメントは丁寧に書き、意図と前提を明確にします。
- 仕様上の前提、数式の意味、数値安定性に関する注意を優先して記述します。

## 4. テスト
- 各ファイル末尾の `if __name__ == "__main__":` で最小テストを実行できるようにします。
- テストは小さく決定的な入力を用い、解が明確なケースを優先します。

## 5. ドキュメント
- 変更が入った場合は、該当する `documents/` 内の文書を同時に更新します。
- モジュールの役割と依存関係は、文書に明示します。
