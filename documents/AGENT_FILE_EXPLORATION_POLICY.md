# Agent ファイル探索順ポリシー

この文書は、Coding Agent が「どのファイルから順に調べるか」を規定します。

## 1. 目的

- 探索の無駄を減らす
- 重要ファイルの見落としを防ぐ
- 編集前に規約・設計を確認する

## 2. 標準探索順

1. スコープ固定
   - `WORKTREE_SCOPE.md`
   - issue/ticket 本文
2. 規約確認
   - `documents/README.md`
   - `documents/coding-conventions-*.md`
3. 設計確認
   - `documents/design/**`
4. 実装（定義側）
   - `python/**/__init__.py`
   - `python/**/core.py`
   - 型・共通ユーティリティ
5. 実装（利用側）
   - `tests/**`
   - 呼び出し元モジュール
6. 編集と検証
   - 小差分編集 → 静的解析/テスト → レポート出力

## 3. 探索優先度スコア

候補ファイルは次のスコアでソートして読む。

$$
score(file) = 0.40\,R + 0.30\,D + 0.20\,T + 0.10\,C
$$

- $R$: 要求文との関連度（キーワード一致）
- $D$: 依存中心性（import / 呼び出し関係）
- $T$: テスト失敗との距離（近いほど高）
- $C$: 変更頻度（最近よく変わるファイル）

## 4. 停止条件

- 検証が通り、変更目的を満たしたら終了
- 3回連続で同種エラーが再発したら人へエスカレーション
- スコープ外ファイルに波及する場合は、先に設計文書更新を行う

## 5. 実行ログ

各 run で次を保存する。

- 探索順リスト
- 読み込んだファイル一覧
- 編集差分
- 検証結果
- 失敗時の反省メモ
