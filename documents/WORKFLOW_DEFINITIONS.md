# プロジェクト10大ワークフロー定義

## 概要

プロジェクトの自動化・標準化を実現する10個の主要ワークフロー。
各ワークフローに対応するSkillを別途実装。

---

## ワークフロー一覧

### 1️⃣ Experiment Initialization (Skill 8)

**目的**: 実験環境の初期化と設定管理  
**入力**: 実験名、ハイパーパラメータ、データセット  
**出力**: 実験ディレクトリ、設定ファイル、ログディレクトリ  
**関連ドキュメント**: `documents/experiment-workflow.md`

| ステップ | 処理内容 |
|---------|--------|
| 1. 検証 | 入力パラメータの妥当性確認 |
| 2. 初期化 | ディレクトリ・ファイル作成 |
| 3. 設定生成 | YAML設定ファイル生成 |
| 4. ログ設定 | ロギング初期化 |
| 5. 確認 | 初期化完了の検証 |

---

### 2️⃣ Hyperparameter Tuning (Skill 9)

**目的**: ハイパーパラメータの自動最適化  
**入力**: 目的関数、パラメータ範囲、最適化戦略  
**出力**: 最適パラメータ、収束プロット、チューニングレポート  
**依存**: Skill 8

| ステップ | 処理内容 |
|---------|--------|
| 1. 準備 | パラメータ空間定義 |
| 2. 探索 | Grid/Bayesian/Random探索 |
| 3. 評価 | 各候補の性能評価 |
| 4. 更新 | パラメータ更新・記録 |
| 5. 収束判定 | 終了条件の確認 |

---

### 3️⃣ Results Analysis & Visualization (Skill 10)

**目的**: 実験結果の多面的分析と可視化  
**入力**: 実験ログ、中間結果、最終結果  
**出力**: 分析レポート、プロット、比較表  
**依存**: Skill 8, 9

| ステップ | 処理内容 |
|---------|--------|
| 1. 抽出 | ログからメトリクス抽出 |
| 2. 集計 | 統計量計算 |
| 3. 比較 | 複数実験の比較 |
| 4. 可視化 | プロット・グラフ生成 |
| 5. 統合 | レポート統合 |

---

### 4️⃣ Report Generation (Skill 11)

**目的**: 実験・分析結果のMarkdownレポート生成  
**入力**: 分析結果、メタデータ、メディア  
**出力**: Markdownレポート、PDF（オプション）  
**依存**: Skill 3, 10

| ステップ | 処理内容 |
|---------|--------|
| 1. テンプレート選択 | レポート形式決定 |
| 2. 本文生成 | Markdownコンテンツ作成 |
| 3. メディア埋込 | グラフ・画像の統合 |
| 4. 参考文献管理 | BibTeX・参照処理 |
| 5. 出力 | ファイル生成 |

---

### 5️⃣ Documentation Sync (Skill 12)

**目的**: コード・設計・結果ドキュメントの同期  
**入力**: コード変更、実験結果、メタデータ  
**出力**: 更新されたMarkdownドキュメント  
**関連**: `documents/DOCSTRING_GUIDE.md`

| ステップ | 処理内容 |
|---------|--------|
| 1. スキャン | コード・実験の変更検出 |
| 2. 抽出 | docstringからドキュメント抽出 |
| 3. 検証 | ドキュメント一貫性確認 |
| 4. 更新 | Markdown更新 |
| 5. リンク確認 | 参照リンク検証 |

---

### 6️⃣ Code Quality Gate (Skill 13)

**目的**: 静的解析・コード品質チェック  
**入力**: コード、ルール定義  
**出力**: 品質スコア、警告リスト、修正提案  
**依存**: Skill 6 (Static Check)

| ステップ | 処理内容 |
|---------|--------|
| 1. 複雑度分析 | Cyclomatic complexity計測 |
| 2. 重複検出 | コード重複箇所抽出 |
| 3. 命名規約 | 命名ルール検査 |
| 4. スメル検出 | アンチパターン抽出 |
| 5. スコア算出 | 総合品質スコア計算 |

---

### 7️⃣ Type & Lint Check (Skill 14)

**目的**: 型安全性・コード形式の厳密化  
**入力**: Pythonコード  
**出力**: 型チェックレポート、Lintエラー、修正提案  
**関連**: `documents/coding-conventions-python.md`

| ステップ | 処理内容 |
|---------|--------|
| 1. Pyright実行 | 型チェック（strict） |
| 2. Flake8実行 | コード形式確認 |
| 3. Black確認 | フォーマット検証 |
| 4. isort確認 | インポート順序検証 |
| 5. 報告 | 統合レポート生成 |

---

### 8️⃣ Test Coverage Monitor (Skill 15)

**目的**: テスト実行と カバレッジ監視  
**入力**: テストスイート、対象コード  
**出力**: カバレッジレポート、テスト結果  
**関連**: `documents/coding-conventions-testing.md`

| ステップ | 処理内容 |
|---------|--------|
| 1. テスト発見 | テストファイル検出 |
| 2. 実行 | pytest実行 |
| 3. カバレッジ計測 | coverage.py実行 |
| 4. 分析 | 未実装パス抽出 |
| 5. 報告 | HTMLレポート生成 |

---

### 9️⃣ Release Management (Skill 16)

**目的**: バージョン管理・リリース プロセス  
**入力**: リリース対象コード、CHANGELOG  
**出力**: タグ、リリースノート、アーティファクト  
**依存**: Skill 14, 15

| ステップ | 処理内容 |
|---------|--------|
| 1. 前置条件確認 | ブランチ・テスト状態確認 |
| 2. バージョン更新 | pyproject.toml更新 |
| 3. CHANGELOG生成 | Git logからCHANGELOG作成 |
| 4. タグ作成 | Git tagコミット |
| 5. リリース | GitHub Releases作成 |

---

### 🔟 Worktree Lifecycle (Skill 17)

**目的**: Git worktree の標準化・自動管理  
**入力**: ブランチ名、作業タイプ  
**出力**: worktree作成、設定、クリーンアップ  
**関連**: `documents/worktree-lifecycle.md`

| ステップ | 処理内容 |
|---------|--------|
| 1. 計画 | worktree命名・配置決定 |
| 2. 作成 | git worktree add実行 |
| 3. 初期化 | Python環境・ツール配置 |
| 4. 作業 | 開発実施 |
| 5. 終了 | worktree削除・クリーンアップ |

---
