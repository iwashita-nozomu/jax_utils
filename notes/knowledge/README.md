# Knowledge Notes

`notes/knowledge/` には、実務で何度も見返す横断的な短い知識メモを置きます。

`notes/themes/` が説明つきの整理ページであるのに対し、こちらは

- パスの解決
- 環境設定
- 実験運用
- Markdown や assets の扱い
- 監視や保存のチェックリスト

を素早く引くための場所です。

## Format

- 1 topic 1 file を基本とします。
- 箇条書きを中心にしてよいです。
- できるだけ特定の実装ではなく、複数 topic にまたがる知識を書きます。
- `path resolution`, `environment setup`, `experiment operations` のような広い主語を使います。
- 詳しい説明は `notes/themes/` や `notes/experiments/` にリンクします。
- 標準的な文献がある topic では、最後に代表文献を置きます。
- 実務メモであっても、文献で支えられる部分は出典を付けます。
