# 日記 — 2026-04-02

## 概要

久しぶりに文書まわりを大きく整理した日でした。
未参照の旧文書、途中集計、重複レビューを削除し、残す文書は入口から辿れる形に整理しました。
そのうえで root の `README.md` を人間向けの作業入口として書き直し、作業開始手順、実験の標準構成、置き場のルールを本文にまとめました。

## 実施内容

- 既存 Markdown 文書の参照関係を棚卸ししました。
  - `documents/`、`reports/`、`reviews/`、`notes/`、`diary/` を横断して未参照候補を洗い出しました。
  - 入口として残す README と、役目を終えた旧文書を分けて確認しました。

- 古い重複文書と一時レポートを削除しました。
  - `documents/` の旧ロードマップ、旧ワークフロー定義、実装サマリを削除しました。
  - `reports/` の途中集計と重複する生成レポートを削除しました。
  - `reviews/` の旧レビューとテスト用レビューを削除しました。

- 入口文書を整理しました。
  - `reports/README.md` を新設しました。
  - `reviews/README.md` を現状に合わせて更新しました。
  - `notes/experiments/README.md`、`notes/themes/README.md`、`notes/worktrees/README.md`、`diary/README.md` に現在の文書一覧を追記しました。

- 規約文と入口文の語句を見直しました。
  - 曖昧な規範表現を減らし、禁止、固定、許可、条件付きの形へ寄せました。
  - 実験運用、review 運用、worktree 運用、agent workflow 周辺の文言を揃えました。

- `README.md` を人間向けに書き直しました。
  - 単なるリンク集ではなく、この README 単体で作業開始できる構成に変更しました。
  - repo の中身、普段の作業手順、実験の標準構成、結果の置き方、各ディレクトリの役割を本文に書きました。

## 確認したこと

- `git diff --check` は通過しました。
- 簡易参照スキャンでは、未参照の非 README / 非テンプレート文書は解消しました。

`origin` 側の bare repo に対する push 自体は成功しましたが、bare repo の GitHub mirror hook は引き続き SSH 鍵不足で失敗しています。
現在の状態は「`origin/main` は更新済み、GitHub mirror は未解決」です。

## 次にやること

- 実験 topic ごとの README も、必要なら同じ方針で読みやすく整える。
